## 嵌入式 linux 系统盘

体系：boot loader + kernal + root file systeam  
实现：uboot + linux + fat32、nfs、ext4...  
体现：uboot.img + kernal.img + rootfs.img 或 文件夹形式

boot loader 的作用是将 kernal 从硬盘中读取到内存里，然后让 cpu 执行 kernal  
kernal 提供任务调度功能以及操作系统的 api，和 rtos 没有本质区别  
root file systeam 是文件系统，存放了给 kernal 用的文件和程序，就是进系统后根目录下的那些东西。可以是 ext4 分区下的文件和文件夹也可以是空闲分区的一个具有 ext4 格式的二进制填充，因为也可以被系统以ext4的方式读取  
这些东西当然可以编译成一个整体的二进制文件，但是没必要，它们可以独立存在、独立开发、独立更新，既然如此，为了维护方便而分开何乐而不为呢

## 开发环境

在 docker 里的 ubuntu 里进行开发

### 创建容器

1. 获取镜像

    ```terminal
    docker pull ubuntu
    ```

3. 创建并运行容器

    ```terminal
    docker run -itd --name ubuntu_container_rk3399 ubuntu
    ```

### 对 ubuntu 容器做一些预操作

1. 进入容器

    ```terminal
    docker exec -it ubuntu_container_rk3399 bash
    ```

> 从 vscode 里的 docker 插件进入，可以方便的用 vscode 编辑容器的文件

2. 更新软件

    ```terminal
    apt update
    apt upgrade
    ```

3. 安装软件

    ```terminal
    apt install git python3 python-pip
    ```

    ```
    ln -s /usr/bin/python3 /usr/bin/python
    pip install formatter
    ```

    > whereis python  
    > rm /usr/bin/python

4. 获取 SDK

    ```
    cd root
    ```

    > 因为容器默认进入根目录

    ```
    repo init --no-clone-bundle --repo-url https://gitlab.com/firefly-linux/git-repo.git -u https://gitlab.com/firefly-linux/manifests.git -b master -m yocto_release.xml
    sync
    ```
