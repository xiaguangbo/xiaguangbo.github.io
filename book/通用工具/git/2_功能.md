## 下载

```terminal
git clone --recursive https://github.com/用户名/仓库名.git
git submodule update --init --recursive
```

`--recursive` 是包括子模块，没有子模块的不需要

`submodule update --init --recursive` 是更新子模块

## 查看远程仓库地址

```terminal
git remote -v
```

## 更改远程仓库地址

1. 设置远程仓库地址

    ```terminal
    git remote set-url origin https://github.com/用户名/仓库名.git
    ```

2. 推送

    ```terminal
    git push origin main
    ```

    `origin`：指代远程仓库
    `main`：分支名

## 同时推送到多个远程仓库

1. 增加远程仓库地址

    - 命令方式

        ```terminal
        git remote set-url --add origin https://github.com/用户名/仓库名.git
        ```

    - 改文件方式

        打开 .git/config 文件，在 [remote "origin"] 下添加多个 `url = https://github.com/用户名/仓库名.git`

2. 同时推送

    ```terminal
    git push -u origin master
    ```

## 强制推送

面对乱七八糟的好方法

```terminal
git push -f origin main
```

## 获取

fetch 常结合 merge 一起用，git fetch + git merge == git pull。一般要用 git fetch + git merge，因为 git pull 会将代码直接合并，造成冲突等无法知道，fetch 代码下来要 git diff origin/branch_name 来看一下差异然后再合并

## 暂存

会把所有未提交的修改（包括暂存的和非暂存的）都保存起来，这部分对仓库不可见，这时`status`也可以看出来无任何情况。stash是本地的，不会通过git push命令上传到git server上。当需要还原出来的时候用`stash pop`。

1. 建立stash

    ```terminal
    git stash
    ```

    可以使用`git stash save "test-cmd-stash"`来为其添加信息

2. 查看stash下来的

    ```terminal
    git stash list
    ```

    会输出“stash@{0}: WIP on master: 049d078 added the index file”之类的，如果多次stash，会有stash@{1} ... stash@{2} ...

3. 还原回去

    ```terminal
    git stash pop
    ```

    将第一个stash（就是stash@{0}）还原并删除其stash缓存
