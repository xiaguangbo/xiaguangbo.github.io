## 提交者的用户信息

这里的信息只是方便知道是谁提交的

```terminal
git config --global user.name "用户名"
git config --global user.email "邮箱"
```

## 开启文件大小写区分

算是历史遗留？

```terminal
git config core.ignorecase false
```

## 下载

```terminal
git clone --recursive https://github.com/用户名/仓库名.git
git submodule update --init --recursive
```

`--recursive` 是包括子模块，没有子模块的不需要

`submodule update --init --recursive` 是更新子模块

## 查看远程仓库地址

```terminal
git remote -v
```

## 更改远程仓库地址

1. 设置远程仓库地址

    ```terminal
    git remote set-url origin https://github.com/用户名/仓库名.git
    ```

2. 推送

    ```terminal
    git push origin master
    ```

## 同时推送到多个远程仓库

1. 增加远程仓库地址

    - 命令方式

        ```terminal
        git remote set-url --add origin https://github.com/用户名/仓库名.git
        ```

    - 改文件方式

        打开 .git/config 文件，在 [remote "origin"] 下添加多个 `url = https://github.com/用户名/仓库名.git`

2. 同时推送

    ```terminal
    git push -u origin master
    ```

## 强制推送

面对乱七八糟的好方法

```terminal
git push -f origin main
```

`origin`是远程，`main`是分支
