## 卷

卷是独立的，可以将容器的后期数据部分与容器分开存储  
卷可以在容器之间以及容器和主机之间共享数据  
卷被多个容器使用时，其中某个容器的更改都会同步到其他容器  
删除卷需要在没有容器使用它的时候，才能使用命令`docker volume rm volume_name`删除

- ls
    
    列出已有卷

    ```
    docker volume ls
    ```

- inspect

    查看卷的详细信息，比如所在路径等

    ```
    docker volume inspect volume_name
    ```

    `volume_name`：卷的名字，执行`docker volume ls`可以看到

## 获取镜像

```terminal
docker pull ubuntu:latest
```

- `ubuntu:latest`：获取 ubuntu 镜像，版本为 latest

## 创建卷

```terminal
docker volume create ubuntu_volume_test
```

- `ubuntu_volume_test`：卷的名称，也可以是路径带名称

## 创建并启动容器

```terminal
docker run -itd --name ubuntu_container_test ubuntu:latest
```

- `-i`：交互式操作
- `-t`：使用终端
- `-d`：不直接进入容器
- `--name ubuntu_container_test`：指定新容器名为 ubuntu_container_test

还可以添加一些更为详细的操作

## 查看所有容器

可以看到容器的 ID、名字等信息

```terminal
docker ps -a
```

- `-a`：显示所有。不加是显示正在运行的容器

## 进入容器

```terminal
docker exec -it ubuntu_container_test bash
```
- `-i`：交互式操作
- `-t`：使用终端
- `bash`：进入容器后要执行的命令

> 使用容器的 ID、名字都有效，其他的命令也是

## 导出容器

在容器里安装了软件，使用导出的镜像或压缩包创建容器就不需要安装之前已经安装过的软件了，已经保存了所有的数据

可在容器运行时使用

### 导出容器为镜像（未打包，保留历史版本）

这是在原来的提交记录上有加了一层，用`docker inspect ubuntu_image_test`查看并划到最底下可以看出来

```terminal
docker commit ubuntu_container_test ubuntu_image_test
```

- `ubuntu_container_test`：容器名
- `ubuntu_image_test`：镜像名

还可以添加一些更为详细的操作

### 导出容器为镜像压缩包（打包，保留历史版本）

```terminal
docker save -o ubuntu_container_test_`date+%Y%m%d`.tar ubuntu_container_test
```

- \`date+%Y%m%d\`：给 .tar 的文件名带上了日期，比如 ubuntu_container_test_20220803.tar

### 导出容器为镜像压缩包（打包，合并，只留下最新版本）

```terminal
docker export -o ubuntu_container_test_`date+%Y%m%d`.tar ubuntu_container_test
```

## 主机与容器的文件交互

### 主机到容器

- 将主机的文件复制到容器
    ```terminal
    docker cp abc.txt ubuntu_container_test:/usr/local
    ```

- 将主机的文件夹复制到容器
    ```terminal
    docker cp /tmp ubuntu_container_test:/usr/local/
    ```

    > 注意 /usr/local/ 最后有 /

- 将主机的文件夹复制到容器，并重命名
    ```terminal
    docker cp /tmp ubuntu_container_test:/usr/local/abc
    ```

    > 注意 /usr/local/abc 最后没有 /

### 容器到主机

- 将容器的文件夹复制到主机
    ```terminal
    docker cp ubuntu_container_test:/usr/local /tmp/
    ```

> 其他的应该同理
