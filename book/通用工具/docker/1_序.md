## 简介

docker 主要用于打包开发环境，避免搭建环境的重复劳动。可以打包已经配置好的环境也可以写 Dockerfile 文件让 docker 自动配置环境

docker 依赖 linux 内核，如果主机不是 linux 系统可以在虚拟机里用，或者用专门为 docker 制作的软件（内部依然是虚拟机，但用起来方便）

docker for windows 会使用 windows 的 WSL/WSL2 安装 linux 虚拟机，docker for windows 提供了一个 windows 和 docker 之间方便交互和管理的工具  
docker 内部默认路径也是在这个虚拟机里，包括下载的镜像、创建的容器等。虽然 windows 的终端可以直接使用`docker images`这类命令，但这也是被软件转到虚拟机里执行的

## 安装

搜索 docker winodws，找到并直接安装。安装的过程中会弹框说要安装一个“适用于x86_64的linux subSysteam”之类的，弹框上有链接，点过去找到并安装，然后继续安装 docker

## 简单尝试

1. 获取镜像

    ```terminal
    docker pull ubuntu:latest
    ```

    - `ubuntu:latest`：获取 ubuntu 镜像，版本为 latest

2. 创建并启动容器

    ```terminal
    docker run -itd --name ubuntu_container_test ubuntu:latest
    ```

    - `-i`：交互式操作
    - `-t`：使用终端
    - `-d`：后台运行。不直接进入容器
    - `--name ubuntu_container_test`：指定新容器名为 ubuntu_container_test

3. 进入容器

    ```terminal
    docker exec -it ubuntu_container_test bash
    ```

    - `-i`：交互式操作
    - `-t`：使用终端
    - `bash`：进入容器后要执行的命令

    > 这时终端切换到 linux 里了

4. 尝试 linux

    现在可以尝试一些 ubuntu linux 的命令了  
    这个 linux 只有基础和重要的命令可以用，比如`echo`、`apt-get update && upgrade`，但安装了软件就可以用更多命令了  
    试试`echo hello`

5. 结束

    1. `exit`：退出终端
    2. `docker stop ubuntu_container_test`：让容器停止运行
