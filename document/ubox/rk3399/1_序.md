## 简介

rk3399 有 usb3.0 5Gbps otg，这点对于可以连接电脑的设备很好，传输数据的速度快  
用的是 firefly-rk3399 板卡

### 启动流程

rk3399 芯片内有 192KByte sram 和 32KByte rom  
启动时首先运行芯片内部预先存储的程序，该程序也称为 bootrom、maskrom，该程序从数个外部存储设备中找“ID Block”信息（查找顺序是 SPI Nor > SPI Nand > eMMC > SD/MMC/TF），找到了就认该设备为启动盘  
bootrom 加载 tpl 到芯片内部的 sram 并执行，tpl 会初始化 dram，结束后回到 bootrom，bootrom 再加载 spl 到 dram 并执行，此时 bootrom 就结束了。spl 会将 u-boot.itb 加载到 dram 中并执行

tpl 因为要初始化 dram 所以包含 dram 的驱动，这样 spl 就不需要驱动了。tpl 和 spl 都来自 uboot

### 启动盘里的数据分布

距离起点的位置：

```txt
sector(512Byte)  : byte                             : file
0x40     d64     : 0x8000     d3_2768      32 KiB : idbloader.img
0x4000   d1_6384 : 0x80_0000  d838_8608    8  MiB : u-boot.itb
0x8000   d3_2768 : 0x100_0000 d1677_7216   16 MiB : boot.img
                 :                                : rootfs.img
```

linux 里制作烧录镜像要先写好所有的分区表信息，再烧录文件。因为 linux 的 fdisk 会清空镜像的前 2048 个扇区，也就是 1MiB  
还需要 GPT分区表，但只有在 u-boot.itb（不包括u-boot.itb）后面的需要添加分区信息。分区信息只需位置和大小，分区里的文件系统由分立的镜像文件决定  
镜像释放后表现为前 16MiB 是空闲，boot.img 里的文件在一个 fat16 分区（fat32 所在的空间不能低于 32MiB），rootfs.img 里的文件在一个 ext4 分区  
如果分区需要文件系统，比如 boot.img，就需要在制作 boot.img 时就将 boot.img 格式化为 fat16，并把对应的文件复制进去。之后就准备个大的 .img 文件，并写好分区表信息，再把各个镜像文件按照要求的位置写到大镜像文件里

- idbloader.img  
    主要包含 ID Block 包头、u-boot-tpl.bin、u-boot-spl.bin  
    这是 rk 芯片的私有格式  

- u-boot.itb  
    主要包含 u-boot-nodtd.bin、bl31.elf、u-boot.dtd，而旧式的 .img 不包含 bl31.elf，也就是 trust.img  
    bl31.elf 包含 TF-A 的 dram、sram、m0 的 .bin 文件。rk3399 内部 有两个 cortex-m0，其一 PMU M0 被 TF-A 使用，其二 Perilp M0 给用户使用

- boot.img  
    主要包含 内核镜像、内核用的设备树

- rootfs.img  
    主要包含 用户使用的程序和文件

移植 ubuntu 到 arm：使用 ubuntu base 镜像挂载到 qemu 虚拟机上，再进去下载各种软件，还有图形界面，之后打包成 rootfs.img 就可以了

## 开发环境

系统镜像的制作可选择使用 yocto、buildroot 自动制作或手动一个一个的制作。windows 下的烧录工具推荐 BalenaEtcher

在 docker 里的 ubuntu 里进行开发

1. 创建并运行容器

    ```
    docker run -itd --privileged --name ubuntu_22_04 ubuntu:22.04
    ```

    -itd：i：可交互。t：交互方式是终端。d：这次不进入容器。i 和 t 必须设置，不然容器无法运行  
    --privileged：可以有 root 权限。该容器默认是 root 用户，但是不加这个依然没有 root 权限  
    ubuntu_22_04：容器名  
    ubuntu:22.04：镜像名和版本
    
    没有镜像会自动下载  

2. 进入容器

    ```
    docker exec -it ubuntu_22_04 bash
    ```

    bash：进入容器后执行 bash，一个终端程序。这样才能正常使用终端

3. 更新软件源

    ```
    apt update
    ```

    本地没有软件源是下载不了软件的

4. 安装软件

    ```
    apt install vim ca-certificates sudo
    ```

    vim：文本编辑器。镜像没有文本编辑器  
    ca-certificates：CA 证书，证明链接是安全的，这个带了包括清华源的证书。清华源需要这个  
    sudo：可以给命令一个临时的 root 权限

5. 换源

    1. 打开文件并删除其全部内容
        ```
        vim /etc/apt/sources.list
        dG
        ```

    2. 复制此处，然后在终端里右击就粘贴了

        ```txt
        deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ focal main restricted universe multiverse
        deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ focal-updates main restricted universe multiverse
        deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ focal-backports main restricted universe multiverse
        deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ focal-security main restricted universe multiverse
        ```

        focal 是 ubuntu 20 的代号，不同的版本有不同的代号，不能搞错了  
        不同版本的系统就更改这个代号就行了，其他的保持原样，代号看看 /etc/apt/sources.list 就知道了

    3. 保存退出
        
        按 esc 到命令模式，再 shift + :，再输入 wq，然后回车

6. 更新软件源和更新本地软件

    ```
    apt update
    apt upgrade
    ```

7. 安装软件

    ```
    apt install sudo
    ```

8. 设置 root 用户密码

    ```
    passwd
    ```

    然后设置密码

    默认是 root 用户，

9. 添加普通用户

    ```
    adduser xiaguangbo
    ```

    xiaguangbo：用户名  

    然后设置密码。剩下的参数使用默认值
    
    普通用户也有作用，比如 yocto 明确不要在 root 用户下运行，不管有没有 root 权限，不然就报错

10. 让用户可以获得 root 权限

    ```
    vim /etc/sudoers
    ```

    找到 root ALL=(ALL) ALL，在下面添加 xiaguangbo ALL=(ALL) ALL，然后强制写入并退出（在命令后面加感叹号），因为该文件是只读权限

11. 切换到普通用户

    ```
    su xiaguangbo
    ```

### 编译出 bl31.elf

这是 TF-A 的一部分。没有这个进不了 linux，这是 ARMv8 架构需要的

1. 安装软件

    ```
    apt install gcc-arm-none-eabi gcc-aarch64-linux-gnu git make
    ```

    rk3399 有个 cortex-m0 要给 TF-A 用，所以需要 gcc-arm-none-eabi 来编译这个 m0 的程序

2. 下载源码

    ```
    cd /root
    git clone https://github.com/ARM-software/arm-trusted-firmware.git
    ```

3. 编译

    ```
    cd arm-trusted-firmware
    make PLAT=rk3399 CROSS_COMPILE=aarch64-linux-gnu-
    ```

    会生成 arm-trusted-firmware/build/rk3399/release/bl31.elf  
    TF-A 对于 rk3399 的适配在 arm-trusted-firmware/plat/rockchip/rk3399

### 编译出 idbloader.img 和 u-boot.itb

1. 安装软件

    ```
    apt install bison flex libssl-dev device-tree-compiler python3
    ```

2. 下载源码

    ```
    cd /root
    git clone https://github.com/u-boot/u-boot.git
    ```

3. 将 bl31.elf 复制到 u-boot 目录下

    ```
    cd u-boot
    cp /root/arm-trusted-firmware/build/rk3399/release/bl31/bl31.elf ./
    ```

4. 编译

    ```
    make firefly-rk3399_defconfig
    make ARCH=arm CROSS_COMPILE=aarch64-linux-gnu-
    ```

    会生成 u-boot/idbloader.img、u-boot/u-boot.itb  
    firefly-rk3399_defconfig 在 u-boot/configs 下

### 编译出 boot.img

1. 编译 linux

    1. 安装软件

        ```
        apt install wget tar xz-utils bc
        ```

    2. 下载 linux 源码

        ```
        cd /root
        wget -c https://cdn.kernel.org/pub/linux/kernel/v5.x/linux-5.19.1.tar.xz
        tar -xJf linux-5.19.1.tar.xz
        ```

    3. 编译 linux

        ```
        cd linux-5.19.1
        make ARCH=arm64 CROSS_COMPILE=aarch64-linux-gnu- defconfig
        make ARCH=arm64 CROSS_COMPILE=aarch64-linux-gnu- -j$(nproc)
        ```

        会生成 linux-5.19.1/arch/arm64/boot/Image 和 linux-5.19.1/arch/arm64/boot/dst/rockchip/*.dtb

2. 得到 boot.img

    1. 安装软件

        ```
        apt install dosfstools
        ```

    2. 收集所需文件

        ```
        mkdir boot_img
        cp arch/arm64/boot/dts/rockchip/rk3399-firefly.dtb boot_img/
        cp arch/arm64/boot/Image boot_img/
        mkdir boot_img/extlinux
        cat>boot_img/extlinux/extlinux.conf<<EOF
        label rockchip-kernel-5.19.1
            kernel /Image
            fdt /rk3399-firefly.dtb
            append earlycon=uart8250,mmio32,0xff1a0000 root=PARTUUID=B921B045-1D rootwait rootfstype=ext4 init=/sbin/init
        EOF
        ```

        会生成 linux-5.19.1/boot_img/*

    3. 生成 boot.img

        ```
        dd if=/dev/zero of=boot.img bs=1M count=64
        mkfs.fat boot.img
        rm -r boot_img_tmp
        mkdir boot_img_tmp
        mount boot.img boot_img_tmp/
        cp -r boot_img/* boot_img_tmp/
        umount boot_img_tmp
        ```

        会生成 linux-5.19.1/boot.img

### 编译出 rootfs.img

1. 安装软件

    ```
    apt install libncurses-dev bzip2
    ```

2. 下载 busybox 源码

    ```
    cd /root
    wget -c https://busybox.net/downloads/busybox-1.35.0.tar.bz2
    tar -jxf busybox-1.35.0.tar.bz2
    ```

3. 编译出 rootfs 的目录

    1. 配置

        ```
        cd busybox-1.35.0
        make CROSS_COMPILE=aarch64-linux-gnu- menuconfig
        ```

    2. 在出现的类图形界面里配置

        ``` menuconfig
        Settings->
            --- Build option
            [*]Build static binary (no shared libs)
        ```

        退出后会弹出是否保存，选择保存

    3. 编译

        ```
        make CROSS_COMPILE=aarch64-linux-gnu- install
        ```

        会生成 busybox-1.35.0/_install/*

4. 完善 rootfs 的目录

    ```
    cd _install
    mkdir etc dev mnt
    mkdir -p proc sys tmp mnt
    mkdir -p etc/init.d/
    cat>etc/fstab<<EOF
    proc        /proc       proc        defaults        0       0
    tmpfs       /tmp        tmpfs       defaults        0       0
    sysfs       /sys        sysfs       defaults        0       0
    EOF
    cat>etc/init.d/rcS<<EOF
    echo -e "Welcome to tinyLinux"
    /bin/mount -a
    echo -e "Remounting the root filesystem"
    mount  -o  remount,rw  /
    mkdir -p /dev/pts
    mount -t devpts devpts /dev/pts
    echo /sbin/mdev > /proc/sys/kernel/hotplug
    mdev -s
    EOF
    chmod 755 etc/init.d/rcS
    cat>etc/inittab<<EOF
    ::sysinit:/etc/init.d/rcS
    ::respawn:-/bin/sh
    ::askfirst:-/bin/sh
    ::cttlaltdel:/bin/umount -a -r
    EOF
    chmod 755 etc/inittab
    cd dev
    mknod console c 5 1
    mknod null c 1 3
    mknod tty1 c 4 1
    ```

    在 busybox-1.35.0/_install/ 下增加了一些文件

5. 编译出 rootfs.img

    ```
    dd if=/dev/zero of=rootfs.img bs=1M count=32
    mkfs.ext4 rootfs.img
    rm -r rootfs_img
    mkdir rootfs_img
    mount -o loop rootfs.img rootfs_img
    cp -rf _install/* rootfs_img
    umount rootfs_img
    ```

    生成了 busybox-1.35.0/rootfs.img

### 打包完整系统

rk3399 的 tf卡 的启动顺序落后于 eMMC，所以要先擦除 eMMC。用个能被工具识别的镜像就可以使用擦除功能了  
启动后显示的 uboot 的建立时间就是刚编译的时间

创建完整镜像在之后一次性烧写

1. 安装软件

    ```
    apt install fdisk
    ```

2. 将所需的文件复制过来

    ```
    cd /root
    mkdir systeam_img
    cd systeam_img
    cp /root/u-boot/idbloader.img ./
    cp /root/u-boot/u-boot.itb ./
    cp /root/linux-5.19.1/boot.img ./
    cp /root/busybox-1.35.0/rootfs.img ./
    ```

3. 打包

    ```
    dd if=/dev/zero of=nano.img bs=1M count=1024

    fdisk nano.img << EOF
    g
    n

    32768
    +112M
    n

    262144
    +50M
    w
    EOF

    dd if=idbloader.img of=nano.img bs=1k seek=32 conv=notrunc
    dd if=u-boot.itb of=nano.img bs=1M seek=8 conv=notrunc
    dd if=boot.img of=nano.img bs=1M seek=16 conv=notrunc
    dd if=rootfs.img of=nano.img bs=1M seek=128
    ```

    g   : GPT 分区表  
    bs  ：指定的块大小，字节为单位，可带额外单位。这个块大小是对于 seek 参数来说的，和物理设备无关  
    seek：从哪开始写入，以 bs 为单位，可带额外单位  
    conv=notrunc：按照写入量进行覆盖，保留后面原来就有的。最后一个没加，就能自动截断后面没用到的部分，这样不管刚开始的 nano.img 有多大都能自动缩小到实际大小
    
    额外单位：w 表示 * 2，b 表示 * 512，k 表示 * 1024，M 表示 * 1024 * 1024，G 表示 * 1024 * 1024 * 1024
