## 简介

rk3399 有 usb3.0 5Gbps otg，这点对于可以连接电脑的设备很好，传输数据的速度快  
用的是 firefly-rk3399 板卡

### 启动流程

rk3399 芯片内有 192KByte sram 和 32KByte rom  
启动时首先运行芯片内部预先存储的程序，该程序也称为 bootrom、maskrom，该程序从数个外部存储设备中找“ID Block”信息（查找顺序是 SPI Nor > SPI Nand > eMMC > SD/MMC/TF），找到了就认该设备为启动盘  
bootrom 加载 tpl 到芯片内部的 sram 并执行，tpl 会初始化 dram，结束后回到 bootrom，bootrom 再加载 spl 到 dram 并执行，此时 bootrom 就结束了。spl 会将 u-boot.itb 加载到 dram 中并执行

tpl 因为要初始化 dram 所以包含 dram 的驱动，这样 spl 就不需要驱动了。tpl 和 spl 都来自 uboot

### 启动盘里的数据分布

距离起点的位置：

```txt
sector(512Byte)   : byte                             : file
0x40     d64      : 0x8000     d3_2768      32 KiB : idbloader.img
0x4000   d1_6384  : 0x80_0000  d838_8608    8  MiB : u-boot.itb
0x8000   d3_2768  : 0x100_0000 d1677_7216   16 MiB : boot.img
0x40000  d26_2144 : 0x800_0000 d1_3421_7728 128MiB : rootfs.img
```

linux 里制作烧录镜像要先写好所有的分区表信息，再烧录文件。因为 linux 的 fdisk 会清空镜像的前 2048 个扇区，也就是 1MiB  
还需要 GPT分区表，但只有在 u-boot.itb（不包括u-boot.itb）后面的需要添加分区信息。分区信息只需位置和大小，分区里的文件系统由分立的镜像文件决定  
镜像释放后表现为前 16MiB 是空闲，boot.img 里的文件在一个 fat16 分区（fat32 所在的空间不能低于 32MiB），rootfs.img 里的文件在一个 ext4 分区  
如果分区需要文件系统，比如 boot.img，就需要在制作 boot.img 时就将 boot.img 格式化为 fat16，并把对应的文件复制进去。之后就准备个大的 .img 文件，并写好分区表信息，再把各个镜像文件按照要求的位置写到大镜像文件里

- idbloader.img  
    主要包含 ID Block 包头、u-boot-tpl.bin、u-boot-spl.bin  
    这是 rk 芯片的私有格式  

- u-boot.itb  
    主要包含 u-boot-nodtd.bin、bl31.elf、u-boot.dtd，而旧式的 .img 不包含 bl31.elf，也就是 trust.img  
    bl31.elf 包含 TF-A 的 dram、sram、m0 的 .bin 文件。rk3399 内部 有两个 Cortex-M0，PMU M0 被 TF-A 使用，Perilp M0 给用户使用

- boot.img  
    主要包含 内核镜像、内核用的设备树

- rootfs.img  
    主要包含 用户使用的程序和文件

## 开发环境

只需要启动到内核即可，根文件系统在电脑上制作并在 qemu 里验证，之后再烧录到 tf卡 里

在 docker 里的 ubuntu 里进行开发

1. 创建并运行容器

    ```
    docker run -itd --privileged --name ubuntu_22_04 ubuntu:22.04
    ```

    -itd：i：可交互。t：交互方式是终端。d：这次不进入容器。i 和 t 必须设置，不然容器无法运行  
    --privileged：可以有 root 权限。该容器默认是 root 用户，但是不加这个依然没有 root 权限  
    ubuntu_22_04：容器名  
    ubuntu:22.04：镜像名和版本
    
    没有镜像会自动下载  

2. 进入容器

    ```
    docker exec -it ubuntu_22_04 bash
    ```

    bash：进入容器后执行 bash，一个终端程序。这样才能正常使用终端

    也可以在 vscode 安装 docker 扩展，右击容器选择 Attach Shell 就能让容器 shell 出现在 vscode 内嵌的终端上

3. 更新软件源

    ```
    apt update
    ```

    本地没有软件源是下载不了软件的

4. 安装软件

    ```
    apt install vim ca-certificates
    ```

    vim：文本编辑器。镜像没有文本编辑器  
    ca-certificates：CA 证书，证明链接是安全的，这个带有清华源的证书。清华源需要这个  

5. 换源

    1. 打开文件并删除其全部内容

        ```
        vim /etc/apt/sources.list
        ```

        全部删除：手动输入 ggdG

    2. 复制此处，然后在终端里右击就粘贴了

        ```txt
        deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ jammy main restricted universe multiverse
        deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ jammy-updates main restricted universe multiverse
        deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ jammy-backports main restricted universe multiverse
        deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ jammy-security main restricted universe multiverse
        ```

        jammy 是 ubuntu 22 的代号，不同的版本有不同的代号，不能搞错了  
        不同版本的系统就更改这个代号就行了，其他的保持原样，代号看看 /etc/apt/sources.list 就知道了

        保存退出：按 esc 到命令模式，再 shift + :，再输入 wq，然后回车

6. 更新软件源和更新本地软件

    ```
    apt update
    apt upgrade
    ```

7. 创建工作目录

    ```
    mkdir ~/ubox
    ```

### 用 buildroot 制作

下载的文件在 dl 下，有的是源码有的是压缩包，比如 arm-trusted-firmware、u-boot

输出的镜像在 output/images 下，有完整镜像和分立的镜像，比如 idbloader.img、u-boot.itb、rootfs.ext4

##### 创建 firefly-rk3399 目录

```
cd ~/ubox/buildroot-2022.08-rc1
mkdir board/firefly/firefly-rk3399
```

##### 添加 extlinux.conf

1. 创建

    ```
    vim board/firefly/firefly-rk3399/extlinux.conf
    ```

2. 内容

    ```txt
    label firefly-rk3399 linux
        kernel /Image
        devicetree /rk3399-firefly.dtb
        append earlycon=uart8250,mmio32,0xff1a0000 root=/dev/mmcblk1p2 rootwait
    ```

    kernel 和 devicetree 指的是编译完成后生成的 output/images 下的文件

##### 添加 genimage.cfg

1. 创建

    ```
    vim board/firefly/firefly-rk3399/genimage.cfg
    ```

2. 内容

    ```txt
    image boot.vfat {
        vfat {
            files = {
                "Image",
                "rk3399-firefly.dtb",
                "extlinux"
            }
        }

        size = 64M
    }

    image sdcard.img {
        hdimage {
            partition-table-type = "gpt"
        }

        partition loader1 {
            in-partition-table = "no"
            image = "idbloader.img"
            offset = 32K
        }

        partition loader2 {
            in-partition-table = "no"
            image = "u-boot.itb"
            offset = 8M
        }

        partition boot {
            image = "boot.vfat"
            offset = 16M
        }

        partition rootfs {
            image = "rootfs.ext4"
        }
    }
    ```

    boot.vfat/vfat/files 指的是指编译完成后生成的 output/images 下的文件

##### 添加 post-build.sh

1. 创建

    ```
    vim board/firefly/firefly-rk3399/post-build.sh
    ```

2. 内容

    ```sh
    #!/bin/sh

    BOARD_DIR="$(dirname $0)"

    install -m 0644 -D $BOARD_DIR/extlinux.conf $BINARIES_DIR/extlinux/extlinux.conf
    ```

    $BOARD_DIR：这应该是和 configs/firefly_rk3399_defconfig 里的 BR2_TARGET_GENERIC_HOSTNAME 有关  
    $BINARIES_DIR：output/images

3. 添加权限

    ```
    chmod 777 board/firefly/firefly-rk3399/post-build.sh
    ```

##### 添加 ubox_defconfig

1. 创建

    ```
    vim configs/firefly_rk3399_defconfig
    ```

2. 内容

    ```txt
    # Architecture
    BR2_aarch64=y
    BR2_cortex_a72_a53=y

    # Firmware
    BR2_TARGET_ARM_TRUSTED_FIRMWARE=y
    BR2_TARGET_ARM_TRUSTED_FIRMWARE_CUSTOM_GIT=y
    BR2_TARGET_ARM_TRUSTED_FIRMWARE_CUSTOM_REPO_URL="/root/ubox/src_git/arm-trusted-firmware"
    BR2_TARGET_ARM_TRUSTED_FIRMWARE_CUSTOM_REPO_VERSION="origin/master"
    BR2_TARGET_ARM_TRUSTED_FIRMWARE_PLATFORM="rk3399"
    BR2_TARGET_ARM_TRUSTED_FIRMWARE_BL31=y
    BR2_TARGET_ARM_TRUSTED_FIRMWARE_NEEDS_ARM32_TOOLCHAIN=y
    BR2_TARGET_ARM_TRUSTED_FIRMWARE_IMAGES=""

    # Bootloader
    BR2_TARGET_UBOOT=y
    BR2_TARGET_UBOOT_BUILD_SYSTEM_KCONFIG=y
    BR2_TARGET_UBOOT_CUSTOM_GIT=y
    BR2_TARGET_UBOOT_CUSTOM_REPO_URL="/root/ubox/src_git/u-boot"
    BR2_TARGET_UBOOT_CUSTOM_REPO_VERSION="origin/master"
    BR2_TARGET_UBOOT_BOARD_DEFCONFIG="firefly-rk3399"
    BR2_TARGET_UBOOT_NEEDS_DTC=y
    BR2_TARGET_UBOOT_NEEDS_PYTHON3=y
    BR2_TARGET_UBOOT_NEEDS_ATF_BL31=y
    BR2_TARGET_UBOOT_NEEDS_ATF_BL31_ELF=y
    BR2_TARGET_UBOOT_FORMAT_CUSTOM=y
    BR2_TARGET_UBOOT_FORMAT_CUSTOM_NAME="u-boot.itb"
    BR2_TARGET_UBOOT_SPL=y
    BR2_TARGET_UBOOT_SPL_NAME="idbloader.img"

    # Kernel
    BR2_LINUX_KERNEL=y
    BR2_LINUX_KERNEL_CUSTOM_GIT=y
    BR2_LINUX_KERNEL_CUSTOM_REPO_URL="/root/ubox/src_git/linux"
    BR2_LINUX_KERNEL_CUSTOM_REPO_VERSION="origin/master"
    BR2_LINUX_KERNEL_USE_ARCH_DEFAULT_CONFIG=y
    BR2_LINUX_KERNEL_DTS_SUPPORT=y
    BR2_LINUX_KERNEL_INTREE_DTS_NAME="rockchip/rk3399-firefly"
    BR2_LINUX_KERNEL_NEEDS_HOST_OPENSSL=y

    # Filesystem
    BR2_TARGET_GENERIC_HOSTNAME="firefly-rk3399"
    BR2_TARGET_GENERIC_ISSUE="Welcome to firefly-rk3399"
    BR2_TARGET_ROOTFS_EXT2=y
    BR2_TARGET_ROOTFS_EXT2_4=y
    BR2_PACKAGE_HOST_DOSFSTOOLS=y
    BR2_PACKAGE_HOST_GENIMAGE=y
    BR2_PACKAGE_HOST_MTOOLS=y
    BR2_PACKAGE_HOST_UBOOT_TOOLS=y
    BR2_ROOTFS_POST_IMAGE_SCRIPT="support/scripts/genimage.sh"
    BR2_ROOTFS_POST_SCRIPT_ARGS="-c board/firefly/firefly-rk3399/genimage.cfg"
    BR2_ROOTFS_POST_BUILD_SCRIPT="board/firefly/firefly-rk3399/post-build.sh"
    ```

    BR2_TARGET_ARM_TRUSTED_FIRMWARE_PLATFORM：TF-A 对于 rk3399 的适配在 arm-trusted-firmware/plat/rockchip/rk3399  
    BR2_TARGET_UBOOT_BOARD_DEFCONFIG：文件是 u-boot/configs/firefly-rk3399_defconfig  
    BR2_LINUX_KERNEL_INTREE_DTS_NAME：文件是 linux-5.19.1/arch/arm64/boot/dts/rockchip/rk3399-firefly.dts
