## 在 docker 里的 linux 里进行开发
  
windows 安装 docker desktop 时会提示要安装一个适用于 wsl 的包，手动安装后继续安装 docker desktop  
docker 命令需要软件启动后才能执行

1. 创建容器

    ```
    docker create -it --privileged --name ubuntu_container ubuntu:18.04
    ```

    -it：i：可交互。t：交互方式是终端
    --privileged：用户可以有 root 权限

2. 运行容器

    ```
    docker start ubuntu_container
    ```

3. 进入容器

    ```
    docker exec -it ubuntu_container bash
    ```

    bash：进入容器后执行 bash，一个终端程序。这样才能正常使用终端

4. 更新软件源

    ```
    apt update
    ```

    搜索包：sudo apt-cache search packet_name

5. 安装软件

    ```
    apt install vim ca-certificates sudo
    ```

    ca-certificates：CA 证书，证明链接是安全的，这个带了包括清华源的证书。清华源需要这个

6. 换源

    将 /etc/apt/sources.list 内容换成下面的：
    ```txt
    deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ bionic main restricted universe multiverse
    deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ bionic-updates main restricted universe multiverse
    deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ bionic-backports main restricted universe multiverse
    deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ bionic-security main restricted universe multiverse
    ```

7. 更新软件源和更新本地软件

    ```
    apt update
    apt upgrade
    ```

8. 设置 root 用户密码

    ```
    passwd
    ```

    然后设置密码

    默认是 root 用户，

9. 添加普通用户

    ```
    adduser xiaguangbo
    ```

    xiaguangbo：用户名  

    然后设置密码，再一路回车

10. 让用户可以获得 root 权限

    ```
    vim /etc/sudoers
    ```

    找到 root ALL=(ALL:ALL) ALL，在下面添加 xiaguangbo ALL=(ALL:ALL) ALL，然后强制写入

11. 切换到普通用户

    ```
    su xiaguangbo
    ```

### 准备软件

1. 安装软件

    ```
    sudo apt install repo git ssh make gcc libssl-dev liblz4-tool expect g++ patchelf chrpath gawk texinfo chrpath diffstat binfmt-support qemu-user-static live-build bison flex fakeroot cmake gcc-multilib g++-multilib unzip device-tree-compiler ncurses-dev
    ```

    需要补充的软件包：
    ```
    p7zip bc time rsync
    ```

### 准备文件

1. 将下载的文件复制到 ubuntu 里

    ```
    docker cp C:\xiaguangbo\project\arm_rk3588\sdk_linux ubuntu_container:/home/xiaguangbo/
    ```
    
    在 windows 下执行

2. 解压、更新

    ```
    cd ~/sdk_linux
    sudo 7zr x rk3588_repo_sdk_v0.1.0a.7z
    cd rk3588
    sudo .repo/repo/repo sync -l
    sudo .repo/repo/repo start rk3588/firefly --all
    sudo cp ../bundle/* ./
    sudo chmod 777 ./update.sh
    sudo ./update.sh
    ```

    update.sh 里的 7z 命令要改成 7zr

### 编译

1. 配置

    ```
    cd ~/sdk_linux/rk3588
    sudo mkdir rockdev
    sudo ./build.sh itx-3588j-buildroot.mk
    ```

2. 编译

    ```
    sudo ./build.sh
    ```

    编译后的镜像在 /home/xiaguangbo/sdk_linux/rk3588/rockdev/pack 里
