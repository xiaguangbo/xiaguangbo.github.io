## 简介

简单、明了、安全、快速

程序可分为“必须连续运行”和“可独立运行”两部分，可独立运行的部分可优化成多任务，可再优化成多核运行。对程序而言串行和并行没有本质的区别，结果是一样的。多任务时，某个任务需要等待其他程序传递一些信息时该任务才会原地打转，但并不影响整体，因为多任务是轮转运行或并行的。  
cpu 和 fpga 的程序对于连续执行是一样的。面对多任务，单核 cpu 只能轮转，而多核 cpu 可以并行，fpga 的多任务可以是多个功能电路块。  
也就是说，通用语法完全是可行的。虽然面对不同的使用场景会出现不同的使用方法，比如面对属性描述的场景时就只有变量了。

从语法逻辑来看，目前的编程语言可分为三类，分别是顺序执行（C/C++、C#、Rust、JavaScript、Python、Bash、CMake 等）、顺序和并行执行并存（Verilog HDL 等）、属性描述（JSON、HTML、CSS 等）。显然，需要三种语言以满足需求，但他们的语法并不是大相径庭的，比如 JSON 是 JavaScript 的子集，JavaScript 很像 C，还比如 Verilog HDL 是用于 FPGA 的语言，各逻辑语句、功能块之间可以连接以并行运行，自动触发，无需像顺序执行的语言那样通过通知或手动调用，但其语法依然很像 C，只是添加了更多的符合以表示更多的功能。可以有一个完整的虚拟语法然后分出这三种类型的语言

规则简单明了才能通用且快速，拒绝概念混杂，拒绝概念漏洞才能安全

参考特性有内存引用计数式释放、并发、模块、导入导出等

## 分支

```
取其精华，去其糟粕

软件类。C、C++、C#、Rust、Python、Bash：.uc
    需要综合更好的特性和更好的人性化，并去掉毫无意义的东西。静态语言和动态语言没有本质区别

电路类。Verilog HDL、Spinal HDL：.uhdl
    都差不多

配置类。类JSON、DTS、HTML、CSS：.ujson
    都差不多。目的是让程序知道应该配置些什么东西，参数是什么。没有关键字，重点是语法结结构

编译规则类。Makefile、CMake、BitBake、Yocto：.umk
    能用，但一团糟

文档类。Markdown：.umd
    部分关键字符会和写的内容混淆，导致渲染结果不是自己想要的。因为 Markdown 用的关键字符过于普通，很容易就和自己写的内容相同。
    而且标准定义不清晰，为了支持更多的功能而出现了其他渲染插件和类似 Markdown 的东西。
```
