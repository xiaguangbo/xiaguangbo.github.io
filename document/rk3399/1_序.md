## 简介

rk3399 芯片内有 192KiB sram 和 32KiB rom  
启动时首先运行芯片内部预先存储的程序，该程序也称为 bootrom、maskrom，该程序从数个外部存储设备中找“ID Block”信息（查找顺序是 SPI Nor > SPI Nand > eMMC > SD/MMC/TF），找到了就认该设备为启动盘

启动盘里的数据距离起点的位置：
```txt
sector(512Byte)   : byte                           : file
0x40     d64      : 0x8000     d3_2768      32 KiB : idbloader.img
0x4000   d1_6384  : 0x80_0000  d838_8608    8  MiB : u-boot.itb
0x8000   d3_2768  : 0x100_0000 d1677_7216   16 MiB : boot.img
0x40000  d26_2144 : 0x800_0000 d1_3421_7728 128MiB : rootfs.img（也可以紧挨在 boot.img 后面）
```

linux 里制作烧录镜像要先写好所有的分区表信息，再烧录文件。因为 linux 的 fdisk 会清空镜像的前 2048 个扇区，也就是 1MiB  
还需要 GPT分区表，但只有在 u-boot.itb（不包括u-boot.itb）后面的需要添加分区信息。分区信息只需位置和大小，分区里的文件系统由分立的镜像文件决定，比如 rootfs 分区的格式由 rootfs.img 本身的文件系统类型绝定  
镜像释放后表现为前 16MiB 是空闲，boot.img 里的文件在一个 fat16 分区（fat32 所在的空间不能低于 32MiB），rootfs.img 里的文件在一个 ext4 分区  
如果分区需要文件系统，比如 boot.img，就需要在制作 boot.img 时就将 boot.img 格式化为 fat16，并把对应的文件复制进去。之后就准备个大的 .img 文件，并写好分区表信息，再把各个镜像文件按照要求的位置写到大镜像文件里

- idbloader.img：包含 idblock.img、u-boot-tpl.bin、u-boot-spl.bin
- u-boot.itb：包含 u-boot-nodtd.bin、bl31.elf、u-boot.dtd
- boot.img：包含内核镜像、内核用的设备树
- rootfs.img：包含用户使用的程序和文件

## 开发环境

### 在 docker 里的 linux 里进行开发
  
windows 安装 docker desktop 时会提示要安装一个适用于 wsl 的包，手动安装后继续安装 docker desktop  
docker 命令需要软件启动后才能执行

1. 创建容器

    ```
    docker create -itd --privileged --name archlinux_container archlinux
    ```

    -itd：i：可交互。t：交互方式是终端。d：不直接进入容器。i 和 t 必须设置，不然容器无法运行  
    --privileged：可以有 root 权限。该容器默认是 root 用户，但是不加这个依然没有 root 权限  

2. 运行容器

    ```
    docker start archlinux_container
    ```

    停止容器：docker stop archlinux_container

3. 进入容器

    ```
    docker exec -it archlinux_container bash
    ```

    bash：进入容器后执行 bash，一个终端程序。这样才能正常使用终端

    退出 bash：exit

4. 添加软件仓库

    在 /etc/pacman.d/mirrorlist 第一行添加：

    ```
    Server = https://mirrors.tuna.tsinghua.edu.cn/archlinux/$repo/os/$arch
    ```

5. 更新本地软件仓库信息并更新所有软件

    ```
    pacman -Sy
    pacman -Syu
    ```

    pacman -S 软件名: 安装软件  
    pacman -Ss 关键字: 在软件仓库中搜索含关键字的软件包  
    pacman -Qs 关键字: 搜索已安装的软件包  
    pacman -Ru 软件名: 删除软件，同时删除不需要的依赖  

    搜索命令以获取包名：https://man.archlinux.org/man/rpcgen.1

### 手动构建（ubuntu下操作）

#### 生成 bl31.elf

这是 TF-A 的一部分

1. 安装软件

    ```
    apt gcc-arm-none-eabi gcc-aarch64-linux-gnu git make
    ```

    rk3399 有个 cortex-m0 要给 TF-A 用，所以需要 gcc-arm-none-eabi 来编译这个 m0 的程序

2. 下载源码

    ```
    git clone --depth=1 https://github.com/ARM-software/arm-trusted-firmware.git
    ```

3. 生成

    ```
    cd arm-trusted-firmware
    make PLAT=rk3399 CROSS_COMPILE=aarch64-linux-gnu-
    ```

    会生成 arm-trusted-firmware/build/rk3399/release/bl31/bl31.elf  
    
    TF-A 对于 rk3399 的适配在 arm-trusted-firmware/plat/rockchip/rk3399

#### 生成 idbloader.img 和 u-boot.itb

1. 安装软件

    ```
    apt install bison flex libssl-dev device-tree-compiler python3
    ```

2. 下载源码

    ```
    cd ..
    git clone https://github.com/u-boot/u-boot.git
    ```

3. 将 bl31.elf 复制到 u-boot 目录下

    ```
    cp arm-trusted-firmware/build/rk3399/release/bl31/bl31.elf u-boot/
    ```

4. 生成

    ```
    cd u-boot
    make firefly-rk3399_defconfig
    make ARCH=arm CROSS_COMPILE=aarch64-linux-gnu-
    ```

    会生成 u-boot/idbloader.img、u-boot/u-boot.itb  
    
    firefly-rk3399_defconfig 在 u-boot/configs 下

#### 生成 boot.img

1. 编译 linux

    1. 安装软件

        ```
        apt install wget tar xz-utils bc
        ```

    2. 下载 linux 源码

        ```
        cd ..
        wget -c https://cdn.kernel.org/pub/linux/kernel/v5.x/linux-5.19.2.tar.xz
        tar -xJf linux-5.19.2.tar.xz
        ```

    3. 编译 linux

        ```
        cd linux-5.19.2
        make ARCH=arm64 CROSS_COMPILE=aarch64-linux-gnu- defconfig
        make ARCH=arm64 CROSS_COMPILE=aarch64-linux-gnu- -j$(nproc)
        ```

        会生成 linux-5.19.1/arch/arm64/boot/Image 和 linux-5.19.1/arch/arm64/boot/dts/rockchip/*.dtb

2. 生成

    1. 安装软件

        ```
        apt install dosfstools
        ```

    2. 收集所需文件

        ```
        mkdir boot_img
        cp arch/arm64/boot/dts/rockchip/rk3399-firefly.dtb boot_img/
        cp arch/arm64/boot/Image boot_img/
        mkdir boot_img/extlinux
        cat>boot_img/extlinux/extlinux.conf<<EOF
        label rockchip-kernel-5.19.2
            kernel /Image
            fdt /rk3399-firefly.dtb
            append earlycon=uart8250,mmio32,0xff1a0000 root=/dev/mmcblk1p2 rootwait rootfstype=ext4
        EOF
        ```

        会生成 linux-5.19.1/boot_img/*

        extlinux.conf 的内容可以在 buildroot 里找到

    3. 生成

        ```
        rm -rf boot_img_mount
        dd if=/dev/zero of=boot.img bs=1M count=64
        mkfs.fat boot.img
        mkdir boot_img_mount
        mount boot.img boot_img_mount/
        cp -r boot_img/* boot_img_mount/
        umount boot_img_mount
        ```

        会生成 linux-5.19.1/boot.img

        boot_img/*：不加 * 会把 boot_img 放到 boot_img_tmp 里，加 * 是复制 boot_img 下的文件

3. 脚本

    ```
    #!/bin/bash 
    mnt () 
    { 
        echo "MOUNTING" 
        mount -t proc /proc ${2}/proc 
        mount -t sysfs /sys ${2}/sys 
        mount -o bind /dev ${2}/dev 
        mount -o bind /dev/pts ${2}/dev/pts 
        chroot ${2} 
    }
    umnt () 
    { 
        echo "UNMOUNTING" 
        umount ${2}/proc 
        umount ${2}/sys 
        umount ${2}/dev/pts 
        umount ${2}/dev 
    } 
    
    if [ "$1" = "-m" ] && [ -n "$2" ]; 
    then 
        mnt $1 $2 
        echo "mnt -m pwd" 
    elif [ "$1" = "-u" ] && [ -n "$2" ]; 
    then 
        umnt $1 $2 
        echo "mnt -u pwd" 
    else 
        echo "" 
        echo "Either 1'st, 2'nd or bothparameters were missing" 
        echo "" 
        echo "1'st parameter can be one ofthese: -m(mount) OR -u(umount)" 
        echo "2'nd parameter is the full pathof rootfs directory(with trailing '/')" 
        echo "" 
        echo "For example: ch-mount -m/media/sdcard/" 
        echo "" 
        echo 1st parameter : ${1} 
        echo 2nd parameter : ${2} 
    fi
    ```

#### 打包并测试

rk3399 的 tf卡 的启动顺序落后于 eMMC，所以要先擦除 eMMC。用个能被工具识别的镜像就可以使用擦除功能了  
启动后显示的 uboot 的建立时间就是刚编译的时间

创建完整镜像在之后一次性烧写

1. 安装软件

    ```
    apt install fdisk
    ```

2. 将所需的文件复制过来

    ```
    cd ~/project/rockchip_manuel
    mkdir systeam_img
    cd systeam_img
    cp ../u-boot/idbloader.img ./
    cp ../u-boot/u-boot.itb ./
    cp ../linux-5.19.2/boot.img ./
    cp ../rootfs.img ./
    ```

3. 打包

    ```
    dd if=/dev/zero of=nano.img bs=1M count=1024

    fdisk nano.img << EOF
    g
    n

    32768
    +64M
    n

    262144
    +512M
    w
    EOF

    dd if=idbloader.img of=nano.img bs=1k seek=32 conv=notrunc
    dd if=u-boot.itb of=nano.img bs=1M seek=8 conv=notrunc
    dd if=boot.img of=nano.img bs=1M seek=16 conv=notrunc
    dd if=rootfs.img of=nano.img bs=1M seek=128
    ```

    g: GPT 分区表  
    bs：指定的块大小，字节为单位，可带额外单位。这个块大小是对于 seek 参数来说的，和物理设备无关  
    seek：从哪开始写入，以 bs 为单位，可带额外单位  
    conv=notrunc：按照写入量进行覆盖，保留后面原来就有的。最后一个没加，就能自动截断后面没用到的部分，这样不管刚开始的 nano.img 有多大都能自动缩小到实际大小
    
    额外单位：w 表示 * 2，b 表示 * 512，k 表示 * 1024，M 表示 * 1024 * 1024，G 表示 * 1024 * 1024 * 1024
