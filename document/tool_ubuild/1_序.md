## 简介

自动构建系统，使用 C# + .NET 7 开发

会自动记录构建过程，有任何未完成的情况都会在其基础上继续运行

### 启动构建

```
cd rockchip_ubuild
dotnet ubuild/main.cs rockchip/configuration/firefly_rk3399.ub.md
```

### .ub.md 的语法

```
## value

<!-- 变量，值为字符串 -->

tool_path = "../../../tool/"

## rely

<!-- 将这些文件在当前文件的位置执行，并且会等待这些文件执行结束 -->

tool_path + "git/git.ub.md"
tool_path + "make/make.ub.md"

## download

<!-- 会进入 build/download/此文件名（没有 .ub.md 后缀） 执行命令 -->

"git --help"
"cp --help"

## source

<!-- 进入 build/source/此文件名 里执行命令 -->
<!-- 需要复制一份或解压下载的压缩包到这里 -->

"cp --help"
"tar --help"

## patch

<!-- 进入 build/source/此文件名 里执行命令 -->

"patch --help"

## build

<!-- 会进入 build/source/此文件名 里执行命令 -->

"make --help"

## image

<!-- 会进入 build/image 执行命令 -->

"cp --help"
```

## 示例文件的内容

### firefly_rk3399.ub.md

```
## value

tool_path = "../../tool/"

## rely

tool_path + "fdisk/fdisk.ub.md"

"../bootload/uboot/uboot.ub.md"
"../kernel/linux/linux.ub.md"
"../rootfs/busybox/busybox.ub.md"

## image

"cp ../source/u-boot/idbloader.img ./"
"cp ../source/u-boot/u-boot.itb ./"
"cp ../source/linux/boot.img ./"
"cp ../source/busybox/rootfs.img ./"

"
dd if=/dev/zero of=nano.img bs=1M count=1024

fdisk nano.img << EOF
g
n

32768
+64M
n

262144

w
EOF

dd if=idbloader.img of=nano.img bs=1k seek=32 conv=notrunc
dd if=u-boot.itb of=nano.img bs=1M seek=8 conv=notrunc
dd if=boot.img of=nano.img bs=1M seek=16 conv=notrunc
dd if=rootfs.img of=nano.img bs=1M seek=128
"
```
