## 嵌入式 linux 系统盘

体系：boot loader + kernal + root file systeam  
实现：uboot + linux + fat32、nfs、ext4...  
体现：uboot.img + boot.img + rootfs.img 或 分区文件夹形式

boot loader 的作用是将 kernal 从硬盘中读取到内存里，然后让 cpu 执行 kernal。指定位置的 .img  
kernal 提供任务调度功能以及操作系统的 api，和 rtos 没有本质区别。指定位置的 .img 或 fat16 分区下的文件  
root file systeam 是文件系统，存放了给 kernal 用的文件和程序，就是进系统后根目录下的那些东西。可以是 ext4 分区下的文件和文件夹也可以是空闲分区的一个具有 ext4 格式的二进制填充，因为也可以被系统以ext4的方式读取  
这些东西当然可以编译成一个整体的二进制文件，但是没必要，它们可以独立存在、独立开发、独立更新，既然如此，为了维护方便而分开何乐而不为呢

## 开发环境

在 docker 里的 ubuntu 里进行开发

### 创建容器

1. 获取镜像

    ```windows terminal
    docker pull ubuntu:18.04
    ```

    > 不要用其他版本，新版 ubuntu 有很多包不支持

3. 创建并运行容器

    ```windows terminal
    docker run -itd --name ubuntu_container_rk3399 ubuntu:18.04
    ```

### 对 ubuntu 容器做一些预操作

1. 进入容器

    ```windows terminal
    docker exec -it ubuntu_container_rk3399 bash
    ```

    > 建议从 vscode 里的 docker 插件进入，可以方便的用 vscode 编辑容器的文件

2. 换源

    1. 备份

        ```linux terminal
        cp /etc/apt/sources.list /etc/apt/sources.list.bak
        ```

    2. 将 sources.list 的内容改成下面这样

        ```txt
        deb http://mirrors.aliyun.com/ubuntu/ bionic main restricted universe multiverse
        deb http://mirrors.aliyun.com/ubuntu/ bionic-security main restricted universe multiverse
        deb http://mirrors.aliyun.com/ubuntu/ bionic-updates main restricted universe multiverse
        deb http://mirrors.aliyun.com/ubuntu/ bionic-proposed main restricted universe multiverse
        deb http://mirrors.aliyun.com/ubuntu/ bionic-backports main restricted universe multiverse
        deb-src http://mirrors.aliyun.com/ubuntu/ bionic main restricted universe multiverse
        deb-src http://mirrors.aliyun.com/ubuntu/ bionic-security main restricted universe multiverse
        deb-src http://mirrors.aliyun.com/ubuntu/ bionic-updates main restricted universe multiverse
        deb-src http://mirrors.aliyun.com/ubuntu/ bionic-proposed main restricted universe multiverse
        deb-src http://mirrors.aliyun.com/ubuntu/ bionic-backports main restricted universe multiverse
        ```

        > bionic 是 ubuntu 18 的代号，不同的版本有不同的代号，不能搞错了

    3. 更新源

        ```linux terminal
        apt update
        ```

3. 更新软件

    ```linux terminal
    apt update
    apt upgrade
    ```

4. 安装软件

    ```linux terminal

    gcc-aarch64-linux-gnu mtools parted libudev-dev libusb-1.0-0-dev python-linaro-image-tools \
    linaro-image-tools gcc-arm-linux-gnueabihf libssl-dev liblz4-tool genext2fs lib32stdc++6 \
    gcc-aarch64-linux-gnu g+conf autotools-dev libsigsegv2 m4 intltool libdrm-dev curl sed make \
    binutils build-essential gcc g++ bash patch gzip bzip2 perl tar cpio python unzip rsync file bc wget \
    libncurses5 libqt4-dev libglib2.0-dev libgtk2.0-dev libglade2-dev cvs git mercurial rsync openssh-client \
    subversion asciidoc w3m dblatex graphviz python-matplotlib libssl-dev texinfo fakeroot \
    ```

    > 安装过程中 tzdata 会要求设置时区，设置：Asia/Shanghai

5. 处理 Firefly_Linux_SDK

    1. 将下载的文件复制到容器里

        ```windows terminal
        docker cp C:\Users\16175\Downloads\rk3399_linux_release_v2.5.1_20210301_split_dir ubuntu_container_rk3399:/root/
        ```

        > 容器默认是 root 用户，所以这里就放在 /root 下

    2. 解压

        ```linux terminal
        cd /root
        cat rk3399_linux_release_v2.5.1_20210301_split_dir/*firefly_split* | tar -xzv
        ```

        > 这里面很多东西是没有的

    3. 更新

        ```linux terminal
        cd rk3399_linux_release_v2.5.1_20210301
        git config --global --add safe.directory /root/rk3399_linux_release_v2.5.1_20210301/.repo/manifests
        git config --global --add safe.directory /root/rk3399_linux_release_v2.5.1_20210301/.repo/repo
        .repo/repo/repo sync -l
        .repo/repo/repo sync -c --no-tags
        .repo/repo/repo start firefly --all
        ```

        > 会下载很多东西，包括开发文档，很重要

6. 设置编译环境

    1. 在 sdk 根目录下创建 rootfs 镜像目录

        ```linux terminal
        mkdir ubunturootfs
        ```

    2. 将镜像文件复制到容器里

        ```windows terminal
        docker cp C:\Users\16175\Downloads\Ubuntu18.04-Minimal_RK3399PRO_v2.11-56_20220526.img.7z ubuntu_container_rk3399:/root/rk3399_linux_release_v2.5.1_20210301/ubunturootfs/
        ```

    4. 解压

        ```linux terminal
        cd ubunturootfs
        7z x Ubuntu18.04-Minimal_RK3399PRO_v2.11-56_20220526.img.7z
        ```

    6. 设置环境变量

        ```linux terminal
        cd ..
        ./build.sh firefly-rk3399-ubuntu.mk
        export RK_UBOOT_DEFCONFIG=firefly-rk3399
        export LINUX_KERNEL_DEFCONFI=firefly_linux_defconfig
        export RK_KERNEL_DTS=rk3399-firefly
        export RK_PARAMETER=parameter-ubuntu.txt
        export RK_PACKAGE_FILE=rk3399-ubuntu-package-file
        export RK_ROOTFS_IMG=ubunturootfs/Ubuntu18.04-Minimal_RK3399PRO_v2.11-56_20220526.img
        ```

        > firefly-rk3399-ubuntu.mk 在 device/rockchip/rk3399/  
        > RK_UBOOT_DEFCONFIG：uboot 配置文件  
        > LINUX_KERNEL_DEFCONFI：kernel 配置文件  
        > RK_KERNEL_DTS：kernel 用到的 dts  
        > RK_PARAMETER：GPT 分区信息  
        > RK_PACKAGE_FILE：打包配置文件  
        > RK_ROOTFS_IMG：根文件系统镜像路径

    6. 修改 device/rockchip/rk3399/firefly-rk3399-ubuntu.mk

        将该文件里的 RK_ROOTFS_IMG 修改成下面这样

        ```txt
        RK_ROOTFS_IMG=ubunturootfs/Ubuntu18.04-Minimal_RK3399PRO_v2.11-56_20220526.img
        ```

7. 生成 ubuntu 固件

    1. 编译
        ```linux terminal
        ./build.sh kernel
        ./build.sh uboot
        ./build.sh recovery
        ```

    2. 提取

        ```linux terminal
        ./mkfirmware.sh
        ```

    3. 打包

        ```linux terminal
        ./build.sh updateimg
        ```

8. 将固件烧录到 tf卡

    ```windows terminal
    docker cp ubuntu_container_rk3399:/root/rk3399_linux_release_v2.5.1_20210301/rockdev/pack/ C:\Users\16175\Downloads\
    ```

    用 SD_Firmware_Tool 制作启动卡，不是升级卡

    > 这个镜像不是完整镜像，普通烧录工具不行
