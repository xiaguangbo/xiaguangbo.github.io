## 简介

<https://wiki.t-firefly.com/zh_CN/Firefly-RK3399/index.html>  
<https://gitlab.com/firefly-linux>

rk3399 有 usb3.0 5Gbps otg，这点对于可以连接电脑的设备很好，传输数据的速度快

### 启动流程

rk3399 芯片内有 192KByte sram 和 32KByte rom  
启动时首先运行芯片内部预先存储的程序，该程序也称为 bootrom、maskrom，该程序从数个外部存储设备中找“ID Block”信息（查找顺序是 SPI Nor > SPI Nand > eMMC > SD/MMC/TF），找到了就认该设备为启动盘  
bootrom 加载 tpl 到芯片内部的 sram 并执行，tpl 会初始化 dram，结束后回到 bootrom，bootrom 加载 spl 到 dram 并执行，此时 bootrom 就结束了。spl 会将 u-boot.itb 加载到 dram 中并执行

tpl 因为要初始化 dram 所以包含 dram 的驱动，这样 spl 就不需要驱动了  
tpl 和 spl 都来自 uboot

### 启动盘里的数据分布

距离起点的位置：

```txt
sector(512Byte)   : byte                             : file          : partition 
0x40     d64      : 0x8000     d3_2768      32 KByte : idbloader.img : 不建分区
0x4000   d1_6384  : 0x80_0000  d838_8608    8  MByte : u-boot.itb    : 不建分区
0x8000   d3_2768  : 0x100_0000 d1677_7216   16 MByte : boot.img      : 112 MByte(+112M)
0x4_0000 d26_2144 : 0x800_0000 d1_3421_7728 128MByte : rootfs.img    : 剩下的都是
```

linux 里制作烧录镜像要先写好所有的分区表信息，再烧录文件。因为 linux 的 fdisk 会直接清空镜像的前 2048 个扇区（512Byte），也就是 1MByte。推荐用脚本来制作，这样比较方便

- idbloader.img  
    主要包含 ID Block 包头、u-boot-tpl.bin、u-boot-spl.bin  
    这是 rk 芯片的私有格式  

- u-boot.itb  
    主要包含 u-boot-nodtd.bin、bl31.elf、u-boot.dtd  
    bl31.elf 包含 TF-A 的 dram、sram、m0 的 .bin 文件。rk3399 内部 有两个 cortex-m0，其一 PMU M0 被 TF-A 使用，其二 Perilp M0 给用户使用

- boot.img  
    主要包含 内核镜像、内核用的设备树

- rootfs.img  
    主要包含 用户使用的程序和文件

## 嵌入式 linux 系统盘

体系：boot loader + kernal + root file systeam  
实现：uboot + linux + fat32、nfs、ext4...  
体现：uboot.img + boot.img + rootfs.img 或 分区文件夹形式

boot loader 的作用是将 kernal 从硬盘中读取到内存里，然后让 cpu 执行 kernal。指定位置的 .img  
kernal 提供任务调度功能以及操作系统的 api，和 rtos 没有本质区别。指定位置的 .img 或 fat16 分区下的文件  
root file systeam 是根文件系统，是包含了文件的文件系统。存放了给 kernal 用的文件和程序，就是进系统后根目录下的那些东西。可以是 ext4 分区下的文件和文件夹也可以是空闲分区的一个具有 ext4 格式的二进制填充，因为也可以被系统以 ext4 的方式读取  
这些东西当然可以编译成一个整体的二进制文件，但是没必要，它们可以独立存在、独立开发、独立更新，既然如此，为了维护方便而分开何乐而不为呢

### 编译出 bl31.elf

这是 TF-A 的一部分。没有这个进不了 linux，这是 ARMv8 架构需要的

1. 安装软件

    ```linux terminal
    apt install gcc-arm-none-eabi gcc-aarch64-linux-gnu git
    ```

    rk3399 有个 cortex-m0 的程序要编译，所以需要 gcc-arm-none-eabi

2. 下载源码

    ```linux terminal
    cd /root
    git clone https://github.com/ARM-software/arm-trusted-firmware.git
    ```

3. 编译

    ```linux terminal
    cd arm-trusted-firmware
    make PLAT=rk3399 CROSS_COMPILE=aarch64-linux-gnu-
    ```

    会生成 arm-trusted-firmware/build/rk3399/release/bl31.elf  
    TF-A 对于 rk3399 的适配在 arm-trusted-firmware/plat/rockchip/rk3399

### 编译出 idbloader.img 和 u-boot.itb

1. 安装软件

    ```
    apt install bison flex libssl-dev device-tree-compiler
    ```

2. 下载源码

    ```linux terminal
    cd /root
    git clone https://github.com/u-boot/u-boot.git
    ```

3. 将 bl31.elf 复制到 u-boot 目录下

    ```linux terminal
    cp /root/arm-trusted-firmware/build/rk3399/release/bl31/bl31.elf /root/u-boot/
    ```

4. 编译

    ```linux terminal
    make firefly-rk3399_defconfig
    make ARCH=arm64 CROSS_COMPILE=aarch64-linux-gnu-
    ```

    会生成 u-boot/idbloader.img、u-boot/u-boot.itb  
    firefly-rk3399_defconfig 在 u-boot/configs 下

### 启动 uboot

rk3399 的 tf卡 的启动顺序落后于 eMMC，所以要先擦除 eMMC。用个能被工具识别的镜像就可以使用擦除功能了  
启动后显示的 uboot 的建立时间就是刚编译的时间

- 在 tf卡 的指定位置烧写

    1. 烧写 idbloader.img 和 u-boot.itb 到 tf卡

        ```
        dd if=idbloader.img of=sdb seek=64
        dd if=u-boot.itb of=sdb seek=16384
        sync
        ```

        这里假设 sdb 是 tf卡

- 创建完整镜像在之后一次性烧写

    1. 安装软件

        ```linux terminal
        apt install fdisk
        ```

    2. 将所需的文件复制过来

        ```linux terminal
        cd /root
        mkdir systeam_img
        cp /root/u-boot/idbloader.img ./
        cp /root/u-boot/u-boot.itb ./
        ```

    3. 打包

        ```linux terminal
        dd if=/dev/zero of=nano.img bs=1M count=100
        dd if=idbloader.img of=nano.img bs=1k seek=32 conv=notrunc
        dd if=u-boot.itb of=nano.img bs=1M seek=8 conv=notrunc
        ```

        bs  ：指定的块大小，字节为单位，可带额外单位。这个块大小是对于 seek 参数来说的，和物理设备无关  
        seek：从哪开始写入，以 bs 为单位，可带额外单位  
        conv=notrunc：按照写入量进行覆盖，保留后面原来就有的
        
        额外单位：w 表示 *2，b 表示 *512，k 表示 *1024，M 表示 * 1024 * 1024，G 表示 * 1024 * 1024 * 1024

        之后就可以用 BalenaEtcher 进行烧录了

### 编译出 boot.img 和 rootfs.img

1. 安装软件

    ```linux terminal
    apt install wget libncurses-dev
    ```

2. 下载 linux 源码

    ```linux terminal
    cd /root
    wget -c https://cdn.kernel.org/pub/linux/kernel/v5.x/linux-5.19.1.tar.xz
    tar -xJf linux-5.19.1.tar.xz
    ```

3. 编译

    ```linux terminal
    cd linux-5.19.1
    make ARCH=arm64 CROSS_COMPILE=aarch64-linux-gnu- defconfig
    make ARCH=arm64 CROSS_COMPILE=aarch64-linux-gnu- -j$(nproc)
    ```

    会生成 linux-5.19.1/arch/arm64/boot/Image 和 linux-5.19.1/arch/arm64/boot/dst/rockchip/*.dtb

4. 下载 busybox 源码

    ```linux terminal
    cd /root
    wget -c https://busybox.net/downloads/busybox-1.35.0.tar.bz2
    tar -jxf busybox-1.35.0.tar.bz2
    ```

5. 编译

    1. 配置

        ```linux terminal
        cd busybox-1.35.0
        make CROSS_COMPILE=aarch64-linux-gnu- menuconfig
        ```

    2. 在出现的类图形界面里配置

        ```linux terminal menuconfig
        Settings->
            --- Build option
            [*]Build static binary (no shared libs)
        ```

        退出后会弹出是否保存，选择保存

    3. 编译

        ```linux terminal
        make CROSS_COMPILE=aarch64-linux-gnu- install
        ```

        会生成 busybox-1.35.0/_install/*

6. 构建根文件系统目录

    1. 创建脚本

        ```linux terminal
        cd /root/busybox-1.35.0
        touch add_root_dir.sh
        ```

    2. 脚本内容

        ```sh
        #!/bin/sh
        cd _install
        mkdir etc dev mnt
        mkdir -p proc sys tmp mnt
        mkdir -p etc/init.d/
        cat>etc/fstab<<EOF
        proc        /proc       proc        defaults        0       0
        tmpfs       /tmp        tmpfs       defaults        0       0
        sysfs       /sys        sysfs       defaults        0       0
        EOF
        cat>etc/init.d/rcS<<EOF
        echo -e "Welcome to tinyLinux"
        /bin/mount -a
        echo -e "Remounting the root filesystem"
        mount  -o  remount,rw  /
        mkdir -p /dev/pts
        mount -t devpts devpts /dev/pts
        echo /sbin/mdev > /proc/sys/kernel/hotplug
        mdev -s
        EOF
        chmod 755 etc/init.d/rcS
        cat>etc/inittab<<EOF
        ::sysinit:/etc/init.d/rcS
        ::respawn:-/bin/sh
        ::askfirst:-/bin/sh
        ::cttlaltdel:/bin/umount -a -r
        EOF
        chmod 755 etc/inittab
        cd dev
        mknod console c 5 1
        mknod null c 1 3
        mknod tty1 c 4 1
        ```

    3. 执行

        ```linux terminal
        chmod 777 add_root_dir.sh
        ./add_root_dir.sh
        ```

        在 busybox-1.35.0/_install/ 下增加了一些文件

7. 制作根文件系统镜像

    1. 创建脚本

        ```linux terminal
        cd /root/busybox-1.35.0
        touch generate_rootfs.sh
        ```

    2. 脚本内容

        ```sh
        #!/bin/bash
        dd if=/dev/zero of=./rootfs.ext3 bs=1M count=32
        mkfs.ext3 rootfs.ext3
        mkdir fs
        sudo mount -o loop rootfs.ext3 ./fs
        sudo cp -rf ./_install/* ./fs
        sudo umount ./fs
        ```

    3. 执行

        ```linux terminal
        chmod 777 generate_rootfs.sh
        ./generate_rootfs.sh
        ```

        生成了 busybox-1.35.0/rootfs.ext3
