## 简介

<https://wiki.t-firefly.com/zh_CN/Firefly-RK3399/index.html>  
<https://gitlab.com/firefly-linux>

rk3399 有 usb3.0 5Gbps otg，这点对于可以连接电脑的设备很好，传输数据的速度快

### 启动流程

rk3399 芯片内有 192KByte sram 和 32KByte rom  
启动时首先运行芯片内部预先存储的程序，该程序也称为 bootrom、maskrom，该程序从数个外部存储设备中找“ID Block”信息，找到了就认该设备为启动盘  
bootrom 加载 tpl 到芯片内部的 sram 并执行，tpl 会初始化 dram，结束后回到 bootrom，bootrom 加载 spl 到 dram 并执行，此时 bootrom 就结束了。spl 会将 u-boot.itb 加载到 dram 中并执行

tpl 因为要初始化 dram 所以包含 dram 的驱动，这样 spl 就不需要驱动了  
tpl 和 spl 都来自 uboot

### 启动盘里的数据分布

```txt
    sector            : byte                    : file
    0x40     d64      : 0x8000     d3_2768      : idbloader.img  
    0x4000   d1_6384  : 0x80_0000  d838_8608    : u-boot.itb  
    0x8000   d3_2768  : 0x100_0000 d1677_7216   : boot.img  
    0x4_0000 d26_2144 : 0x800_0000 d1_3421_7728 : rootfs.img
```

> 偏移的单位为 扇区（512byte）

- idbloader.img
    主要包含 ID Block 包头、u-boot-tpl.bin、u-boot-spl.bin  
    这是 rk 芯片的私有格式  

- u-boot.itb
    主要包含 u-boot-nodtd.bin、bl31.elf、u-boot.dtd  
    bl31.elf 包含 TF-A 的 dram、sram、m0 的 .bin 文件。rk3399 内部 有两个 cortex-m0，其中一个用于 rk3399 正常工作

- boot.img
    主要包含 内核镜像、内核用的设备树

- rootfs.img
    主要包含 用户使用的程序和文件

## 嵌入式 linux 系统盘

体系：boot loader + kernal + root file systeam  
实现：uboot + linux + fat32、nfs、ext4...  
体现：uboot.img + boot.img + rootfs.img 或 分区文件夹形式

boot loader 的作用是将 kernal 从硬盘中读取到内存里，然后让 cpu 执行 kernal。指定位置的 .img  
kernal 提供任务调度功能以及操作系统的 api，和 rtos 没有本质区别。指定位置的 .img 或 fat16 分区下的文件  
root file systeam 是文件系统，存放了给 kernal 用的文件和程序，就是进系统后根目录下的那些东西。可以是 ext4 分区下的文件和文件夹也可以是空闲分区的一个具有 ext4 格式的二进制填充，因为也可以被系统以 ext4 的方式读取  
这些东西当然可以编译成一个整体的二进制文件，但是没必要，它们可以独立存在、独立开发、独立更新，既然如此，为了维护方便而分开何乐而不为呢

## 开发环境

### docker 

在 docker 里的 ubuntu 里进行开发

1. 创建并运行容器

    ```windows terminal
    docker run -itd --name ubuntu_container_rk3399_latest ubuntu
    ```

    > 没有镜像会自动下载

2. 进入容器

    ```windows terminal
    docker exec -it ubuntu_container_rk3399_latest bash
    ```

    > 建议从 vscode 里的 docker 插件进入，可以方便的用 vscode 编辑容器的文件

3. 换源

    1. 备份

        ```linux terminal
        cp /etc/apt/sources.list /etc/apt/sources.list.bak
        ```

    2. 将 sources.list 的内容改成下面这样

        ```txt
        deb http://mirrors.aliyun.com/ubuntu/ jammy main restricted universe multiverse
        deb http://mirrors.aliyun.com/ubuntu/ jammy-security main restricted universe multiverse
        deb http://mirrors.aliyun.com/ubuntu/ jammy-updates main restricted universe multiverse
        deb http://mirrors.aliyun.com/ubuntu/ jammy-proposed main restricted universe multiverse
        deb http://mirrors.aliyun.com/ubuntu/ jammy-backports main restricted universe multiverse
        deb-src http://mirrors.aliyun.com/ubuntu/ jammy main restricted universe multiverse
        deb-src http://mirrors.aliyun.com/ubuntu/ jammy-security main restricted universe multiverse
        deb-src http://mirrors.aliyun.com/ubuntu/ jammy-updates main restricted universe multiverse
        deb-src http://mirrors.aliyun.com/ubuntu/ jammy-proposed main restricted universe multiverse
        deb-src http://mirrors.aliyun.com/ubuntu/ jammy-backports main restricted universe multiverse
        ```

        > jammy 是 ubuntu 22 的代号，不同的版本有不同的代号，不能搞错了  
        > 不同版本的系统只有更改这个代号就行了，其他的保持原样，代号看看 /etc/apt/sources.list 就知道了

    3. 更新源

        ```linux terminal
        apt update
        ```

4. 更新软件

    ```linux terminal
    apt update
    apt upgrade
    ```

### 编译出 bl31.elf

这是 TF-A 的一部分。没有这个进不了 linux，这是 ARMv8 架构需要的

1. 安装软件

    ```linux terminal
    apt install gcc-arm-none-eabi gcc-aarch64-linux-gnu git
    ```

    > rk3399 有个 cortex-m0 的程序要编译，所以需要 gcc-arm-none-eabi

2. 下载源码

    ```linux terminal
    cd /root
    git clone https://github.com/ARM-software/arm-trusted-firmware.git
    ```

3. 编译

    ```linux terminal
    cd arm-trusted-firmware
    make PLAT=rk3399 CROSS_COMPILE=aarch64-linux-gnu-
    ```

    > 会在 arm-trusted-firmware/build/rk3399/release/bl31 下生成 bl31.elf  
    > TF-A 对于 rk3399 的适配在 arm-trusted-firmware/plat/rockchip/rk3399

### 编译出 idbloader.img 和 u-boot.itb

1. 安装软件

    ```
    apt install bison flex libssl-dev device-tree-compiler
    ```

2. 下载源码

    ```linux terminal
    cd /root
    git clone https://github.com/u-boot/u-boot.git
    ```

3. 将 bl31.elf 复制到 u-boot 目录下

    ```linux terminal
    cp /root/arm-trusted-firmware/build/rk3399/release/bl31/bl31.elf /root/u-boot/
    ```

4. 编译

    ```linux terminal
    make firefly-rk3399_defconfig
    make CROSS_COMPILE=aarch64-linux-gnu-
    ```

    > 会在 u-boot 下生成 idbloader.img 和 u-boot.itb
    >
    > firefly-rk3399_defconfig 在 u-boot/configs 下

### 启动 uboot

- 在 tf卡 的指定位置烧写

    1. 烧写 idbloader.img 和 u-boot.itb 到 tf卡

        ```
        dd if=idbloader.img of=sdb seek=64
        dd if=u-boot.itb of=sdb seek=16384
        sync
        ```

        > 这里假设 sdb 是 tf卡

- 创建完整镜像一次性烧写

    1. 安装软件

        ```linux terminal
        apt install kpartx fdisk
        ```

    2. 烧录到镜像
    
        ```
        cd /root
        mkdir img
        cd img
        dd if=/dev/zero of=lucky.img bs=1M count=300
        dd if=/root/u-boot/idbloader.img of=lucky.img seek=64
        dd if=/root/u-boot/u-boot.itb of=lucky.img seek=16384
        ```
